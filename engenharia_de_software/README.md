1- O primeiro texto traz consigo uma ideia de que engenheiros de software não são tão parecidos assim com desenvolvedores. A colocação "engenheiro" traz a visão de uma posição com um maior domínio de habilidades teóricas, assim como nas engenharias: "civil", "aeronáutica" e "mecânica", onde são de extrema importância a validação teórica antes da prática.

2- No segundo trecho citado, o engenheiro de software possui além do desenvolvivemtno, a também responsabilidade de manter o código funcional e escalável, baseando-se nos três pilares: Mudança, Escala e Trade-Offs.

3- 5 requisitos não funcionais de um código:
3.1- Disponibilidade: O software precisa funcionar coretamente, quando houver uma requisição do usuário, 24/7.
3.2- Performance: O software deve ser performático, não tendo problemas de carregamento excessivo.
3.3- Escalabilidade: O software deve ser desenvolvido pensado em crescer em escala, para que não tenha gargalos por acessos simultâneos, número de usuários excedidos, e etc.
3.4- robustês: O software deve ser robusto, tendo sido desenvolvido visando possíveis erros de usuário ou externos, como: queda de internet, travamentos, por exemplo.
